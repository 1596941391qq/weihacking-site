SEO Audit / Four-Line Strategy

leewow.com SEO 审计与四线策略

这份审计先拆开 leewow.com 的双架构问题——首页是纯 SPA 空壳，产品页是静态 HTML 但内容极薄——再给出一套从首页重建、内容体系、技术治理到站外建设的完整路线。

核心判断

Leewow 的产品页做得比首页好很多：有静态 HTML、有 Schema、有 H1/H2 结构。但首页是一个纯 JS 空壳（10.5KB，0 个链接，0 张图片），Google 爬到的首页只有 meta 标签和一个 loading 动画。站点有 6 种语言的 URL 入口但全指向同一个 canonical，国际化是假的。Sitemap 只有 4 个 URL。内容体系几乎为零。

审计日期：2026-04-21（工具实测版） 技术栈：首页 SPA (React/Vue) + 产品页静态 HTML + nginx/1.26.3 站点类型：AI 产品创造平台（DTC） Sitemap：4 URL（首页 + 产品页 + 隐私 + 条款）
一句话结论

综合评分 42/100。最致命的不是产品页做得差，是首页对 Google 来说基本不存在。一个 10.5KB 的 SPA 空壳带着 4 个 URL 的 sitemap，在 Google 眼里这不是一个网站，是一个着陆页。

Leewow 的产品差异点非常强——"AI Shopify for the Physical World"。但这个差异点还没有长成搜索资产。没有对比页、没有教程、没有博客、没有错误码排查。搜索引擎和用户都看不到这个产品的厚度。
Executive Summary

执行摘要

Leewow 处在一个有意思的矛盾里：产品页的技术 SEO 做得相当好——静态 HTML、SoftwareApplication Schema、H1/H2 结构完整。但首页是一个纯 SPA 空壳，Google 爬到的只有 meta 标签和 loading 动画。更关键的是，站点只有 4 个 URL 在 sitemap 里，内容体系完全空白，没有博客、没有对比页、没有教程。国际化有 6 种语言的 URL 入口（/en /zh /ja /es /fr /de），但全部指向同一个 canonical，对 Google 来说是重复内容而不是多语言。

已经具备的基础

HTTP 正确跳 HTTPS，www 正确跳 non-www，产品页有 Schema 和良好的 H1/H2 结构，OG 标签完整，产品页 TTFB 0.35s 极快。Strict-Transport-Security 已启用。

真正拖后腿的地方

首页是 SPA 空壳（0 个链接、0 张图片）。Sitemap 只有 4 URL。国际化有 URL 入口但 canonical 全部指向英文根域（重复内容）。没有博客生态、没有竞品对比页、没有教程、没有内容角色分工。站外证据层为零。

Priority Findings

关键问题

P0首页空壳

首页是纯 SPA，Google 爬到的只有 meta 标签和 loading 动画

10.5KB 的 HTML 里，0 个链接、0 张图片。body 里只有 loading grid dots 和一个空的 <div id="root">。Google 不执行 JavaScript，爬到的首页就是一个带 meta 的空页。这意味着首页无法向任何子页面传递权重，也无法自己排名任何关键词。

P0Sitemap 极度稀疏

整个站点只有 4 个 URL 在 sitemap 里

首页、1 个产品页、隐私政策、服务条款。大量返回 200 的路径（/shop、/products、/catalog、/docs、/faq、/api、/pricing、/contact）都没有进入 sitemap。Google 只能通过自己爬取发现这些页面，效率极低。

P0国际化重复内容

6 种语言 URL 全部指向同一个 canonical，Google 会当作重复内容

/en、/zh、/ja、/es、/fr、/de 都返回 200，但 canonical 全部指向 https://leewow.com/，没有 hreflang。Google 的处理方式是把它们合并成一个页面，多语言入口完全失效。

P1内容空白

没有博客、没有对比页、没有教程，内容体系完全空白

Leewow 的核心竞品（Printful、Printify、Redbubble）都有数百到数千页内容。Leewow 连 1 篇博客文章都没有。竞品品牌词（"Printful alternative"、"cheaper than Printify"）的截流完全空白。

P1首页 TTFB 过高

首页 TTFB 1.39s，而产品页只有 0.35s

首页是 SPA 需要 JS 启动，TTFB 是产品页的 4 倍。如果首页改造成静态 HTML 或 SSR，TTFB 可以降到 0.4s 以内。

Line 1

在用户搜索竞品和品类词的时候截住他们

替代词 / 对比词 / 品类词 / 礼物词

Leewow 的核心竞品（Printful、Printify、Redbubble、Gelato）已经在搜索结果里经营了多年。正面硬刚它们的品牌词排名很难，但截流是可行的。用户搜"Printful alternative"的时候，脑子里想的不是品牌忠诚，而是"有没有更好的方案"。Leewow 的差异化足够强——不是 print-on-demand，是 AI Product Creation Agent。这个差异点一旦被说清楚，截流效果会很直接。

同时，Leewow 涉及的 15+ 品类（卫衣、T恤、围巾、抱枕、毛毯、鼠标垫等），每一个品类都有长尾搜索词。"custom hoodie AI design"、"AI printed t-shirt"、"personalized blanket gift"——这些查询的竞争度远低于竞品品牌词，而且更接近购买意图。

Line 3

第三条线：技术底盘修复与首页重建

Line 3

先把首页从空壳变成真正的网站入口

首页 SSR / Sitemap / 国际化 / Schema / TTFB

Leewow 的技术底盘呈现有趣的分裂：产品页做得好（静态 HTML + Schema + 0.35s TTFB），但首页是 SPA 空壳（10.5KB + JS 渲染 + 1.39s TTFB）。这种分裂说明产品页是手工制作的高质量页面，而首页是后来的 SPA 应用外壳。

第三条线的核心任务是把首页从 SPA 改造为 SSR/SSG，让 Google 爬到完整的 HTML 内容。同时修复国际化的 canonical 问题和 sitemap 的覆盖问题。

首页重建方案

选项一：保持 SPA 但加预渲染（prerender.io）。选项二：首页改为 SSG 静态页面（像产品页一样）。选项三：全站迁移到 SSR 框架（Nuxt/Next）。推荐选项二，因为产品页已经证明了静态 HTML 的效果。

国际化修复

每个语言版本的 canonical 指向自身 URL。加 hreflang 标签。每个语言版本有独特的 Title 和 Description（不是简单翻译）。

Sitemap 重建

从 4 URL 扩展到完整覆盖：首页 + 产品页 + 品类页 + 对比页 + 教程 + 博客 + FAQ。使用 sitemap index 拆分。加 lastmod 日期。
